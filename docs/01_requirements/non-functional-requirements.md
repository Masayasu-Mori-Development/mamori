## 2. セキュリティ要件

### 2.1. 認証・認可

| 要件 | 内容 |
|------|------|
| 認証方式 | JWT（JSON Web Token） |
| トークン有効期限 | 24時間（アクセストークン）、7日間（リフレッシュトークン） |
| パスワードポリシー | 最低8文字、英数字・記号を含む |
| パスワード保存 | BCryptでハッシュ化（ストレッチング10回以上） |
| 多要素認証（MFA） | 将来実装（MVP後） |

### 2.2. 通信セキュリティ

| 要件 | 内容 |
|------|------|
| 通信プロトコル | HTTPS必須（TLS 1.2以上） |
| SSL証明書 | Let's Encrypt または AWS Certificate Manager |
| CORS設定 | 許可されたオリジンのみアクセス可能 |
| CSRF対策 | トークンベースの検証 |

### 2.3. データ保護

| 要件 | 内容 |
|------|------|
| 機密情報の暗号化 | パスワード、個人情報はハッシュ化/暗号化 |
| データベース暗号化 | AWS RDS暗号化有効化 |
| S3暗号化 | S3バケット暗号化有効化（SSE-S3またはSSE-KMS） |
| ログの取り扱い | 個人情報をログに出力しない |

### 2.4. 脆弱性対策

| 脆弱性 | 対策 |
|--------|------|
| SQLインジェクション | プリペアドステートメント使用 |
| XSS（クロスサイトスクリプティング） | 入力値のエスケープ、Content Security Policy |
| CSRF | トークン検証 |
| セッションハイジャック | HTTPOnly Cookie、Secure属性 |
| 依存関係の脆弱性 | Dependabot/Snykによる定期スキャン |

### 2.5. アクセス制御

| 要件 | 内容 |
|------|------|
| ロールベースアクセス制御 | 施設管理者、スタッフでアクセス権限を分離 |
| マルチテナント分離 | tenant_idによるデータ分離 |
| APIアクセス制限 | 認証済みユーザーのみアクセス可能 |
| 監査ログ | 重要な操作（ログイン、データ変更）をログに記録 |
