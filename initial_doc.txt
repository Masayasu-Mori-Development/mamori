# 保育施設向け監査帳票SaaS 事業構想ドキュメント（MVP版）

# まもり保育ごはん

まもりごはんシリーズ

---

## 1. 目的

保育施設向けに、監査対応帳票を効率化するWeb SaaSを開発する。

### ビジョン

- 給食業務のDX化
- 監査対応の標準化
- 将来的に給食市場全体（介護・病院・学校・社員食堂）へ展開

### 個人目標

- 数億規模の資産形成
- ARR 1億規模を目指す
- 持分を保持しつつスケール可能な構造を作る

---

## 2. 市場戦略

### 初期ターゲット

- 保育施設（関東中心）
- グループ企業内100施設が潜在導入先

### なぜ監査帳票から始めるか

- 必須業務（やらないと怒られる）
- 価格転嫁しやすい
- 解約されにくい
- 差別化しやすい

---

## 3. MVPスコープ

### 実装する機能

- 献立入力
- 栄養計算
- 横浜市帳票PDF出力
- 最低限の施設管理

### 実装しない機能（今はやらない）

- GUI帳票ビルダー
- AI自動帳票生成
- 全自治体対応
- 介護・病院対応
- 高度な分析機能

---

## 4. システム全体像（MVP）

```
[献立入力]
      ↓[共通栄養計算エンジン]
      ↓[自治体テンプレ（固定JSON）]
      ↓
PDF出力
```

---

## 5. 設計思想

### 原則

- 外側は速く作る

- 共通化できる部分は共通化

## 7. 帳票ビルダー構想

### MVP段階

- テンプレはJSONで固定

```json

```

# ② PostgreSQLの特徴

- schemaが強力

# 4. 全体アーキテクチャ

```
Users
  ↓
Route53
  ↓
CloudFront
   ├ hoiku.mamori.jp → S3(hoiku-frontend)
   ├ admin.mamori.jp → S3(admin-frontend)
   └ api.mamori.jp   → ALB
                        ├ /api/core/*  → ECS(core-backend)
                        └ /api/hoiku/* → ECS(hoiku-backend)
                              ↓
                           RDS(PostgreSQL)
                              ↓
                           S3（帳票・ファイル）
```

# 10. ネットワーク設計

## VPC構成

- Public Subnet：ALB
- Private Subnet：ECS / RDS
- RDSは外部公開しない

## セキュリティグループ

- ALB → ECSのみ許可
- ECS → RDSのみ許可
- RDSはPrivateのみ

# 12. 認証設計方針

- 認証はcoreで一元管理
- JWT発行はcore
- hoikuは `/api/hoiku/*` のみ公開
- coreは内部ライブラリとして利用可

---

# 13. BFF方針

- hoikuフロントは `/api/hoiku/*` のみ叩く
- adminフロントは `/api/core/*`
- coreロジックは内部利用のみ
